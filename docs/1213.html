Java FX是用Java创建GUI应用程序的最新和最好的方法。曾经有一段时间，AWT和Swing在用Java开发基于GUI的应用程序时非常流行，但慢慢地，Java Swing让位于 C# ，后者成为开发桌面GUI的标准语言，AWS也输掉了与Swing的战斗。

很长一段时间，没有一种流行的技术来用Java构建GUI应用程序，但后来Java引入了 JavaFX、一种更现代的GUI开发API来收复失地。

虽然我认为Java FX还没有达到Swing已经达到的高度，也没有挑战桌面GUI开发中的 C# ，但是它发展得很好，现在已经成为GUI开发的标准Java技术，尤其是开发跨平台GUI应用程序。

虽然C#非常适合开发基于Windows的GUI，但它仍然不是开发跨平台GUI的解决方案。我知道这两个。NET 和 C# 正在朝那个方向发展，但是前面还有很长的路要走，如果你现在想开发一个跨平台的GUI，Java FX可能是比任何其他技术更好的选择。

JavaFX是Java进化的下一步。它旨在为企业业务应用程序提供一个轻量级的硬件加速平台。

从JDK 7开始，JavaFX自动包含在标准的JDK和JRE包、Java开发工具包JDK和Java运行时环境JRE中。
