术语“设计模式”在这里发挥了作用。简而言之，设计模式是常见编程问题的可靠解决方案。例如，创造性设计模式解决了对象创建问题。

它们已经存在很长时间了，但是Erich Gamma、John Vlissides、Ralph Johnson和Richard Helm在1994年出版的经典著作Ddesign Patterns:Elements of Reusable Object-Oriented Software(通常被称为GOF设计模式)使它们流行起来。这本书详细介绍了24种用于解决常见编程问题的可重用设计模式。

设计模式最初是在 C++ 和Smalltalk中引入的，但它们随后被几乎所有编程语言采用，包括Java、C++、 PHP ，甚至不是严格面向对象的计算机语言，如 JavaScript 。

设计模式的优势是不可估量的；它们可以在各种框架、库和编程语言中找到。
